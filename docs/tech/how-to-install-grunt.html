How to install Grunt

This is up to date with Grunt v0.4.5. Nodejs related projects are fiercely developed and change quickly. If a newer version is out, use that documentation as the final say.

Grunt is the nodejs powered task runner. It is widely supported and incredibly useful.
  1. Install NVM
    • Node Version Manager is the best way to install nodejs.  Do not use aptidude, yum or similar to install nodejs as Linux repositiories are notoriously out of date. NVM is the solution.
    • The different ways to install it are well documented here: https://github.com/creationix/nvm
  2. Install Grunt CLI
    • Procedure is documented here: http://gruntjs.com/getting-started
    • npm install -g grunt-cli
    • This makes Grunt available on the server. The Grunt task runner is installed in any subfolder project using NPM, so we do that next.
  3. Install Grunt Task Runner using package.json
    • Create package.json for project dependencies (https://www.npmjs.org/doc/package.json.html). The Grunt Task runner is included.

    • To install node modules run npm install in the same directory as package.json file.

    • npm install
  4. Grunt Tasks using Gruntfile.js
    • Grunt tasks are contained in a Gruntfile.js file. The format is well documented: http://gruntjs.com/getting-started
    • Tasks are run relative to this file. This is important if you need to create a cron tab or alias for a task. For example, an alias to process less files to css files would be similar to this:
    • alias p1build="pushd '/path/to/gruntfilejs; grunt build; popd;'"
